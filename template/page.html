Lesson 1 - Summary
How the web works
The web is formed of millions, yes you read it correctly, millions of pages written in HTML that are interconnected. These files are stored on servers and interpreted with the aid of browsers, which also display them.
HTML
HTML or Hypertext Markup Language stands at the basis of all the web pages that you access daily. This language helps the computer understand what you are trying to tell it to do.

HTML is formed of a series of tags and elements. You can consider HTML just like you would any other language and these tags are the equivalent of letters from the alphabet. Lesson 4 - Summary
What is CSS?
CSS or Cascading Style Sheet is a programming language that gives style to our HTML and ensures that we avoid unnecessary repletion.
Why should we avoid repetition/ add CSS?
Repetition results in many more lines of code, which makes it more difficult to follow, read and revise.
How do we add CSS?
There are different methods of applying CSS to our HTML.
  • We can either add the "style" tag inside the header.
  • Or we can create a separate CSS file for style and link to it in the header section of our page.
  • Or we can add it inline with the rest of our code, but this is not recommended, as it does not fulfill its purpose of avoiding repetition.

Introduction to Object Oriented Programming
Lesson 1 - Summary
Object Oriented Programming
Object Oriented Programming is a programing model that deals with objects as a whole, that manipulates objects as entities rather than a series of inputs, procedures and outputs. More information can be found here. OOP is used to avoid repetition while writing code and also makes it easier to make changes to an already written program when something needs to be modified or new features need to be added. It is particularly useful when used for large projects due to its modularity. Functions with the same purpose are grouped together; they are easier to move around and that makes it possible for more people to work on different parts of a project. Also, this ability to move object around and not having to worry about the details and only about their implementation can lead to faster software development.
Abstraction
Through abstraction we are able to treat the modules used in OOP as black-boxes as we do not need to be concerned with all the characteristics and/or how all the functions encapsulated in the module work.
Use of functions
Each module contains a series of functions that makes it possible to perform certain tasks. There are predefined functions that are part of the standard library of the programming language that we are using e.g The Python Standard Library for Python. Details about these functions can be found in the documentation, but we do not need to know all the details about the functions in order to use them as they behave the same way each time they are being used.

Lesson 1 - Summary
Networks
A network is a group of entities that can communicate with one another despite the fact that they are not directly connected. The concept is not new though, networks have been referenced in Homer’s work The Iliad.
How can we measure networks?
Networks can be measured in terms of latency or bandwidth. Latency refers to the time it takes the message to get from one source to the destination and it is measured in miliseconds. Bandwidth refers to the amount of information that can be transmitted per unit time and it is measured in bits per second.
What is a bit?
A bit is the smallest unit of information.
What makes a network?
A network must be able to encode and interpret messages, it requires a way of routing the messages as well as some rules that decide who gets to use what resources.
What are protocols?
Protocols are set of rules that decide how people can talk to each other. For example, the protocol used on the web that makes the communication between a web browser and a server possible is HTTP, which stands for Hypertext Transfer Protocol.
GET and POST methods
GET is a method used by the client to make a request to the server and that response is sent back through the POST method. For example, when a person using their browser wants to access some content of a webpage they use the GET method to make the request and the server makes it possible for the browser to display the content by sending it to the browser though the POST method.
Lesson 2 - Summary
URLs

URLs or Uniform Resource Locator is formed out of a protocol, a host which can be either a domain name or an IP and the path which is what we are trying to fetch.

Example: “htttp://www.example.com/”

http is the protocol, www.example.com is the domain and what follows after “/” is the path.

Query parameters (GET parameters)

The parameters are the extra information the server gets when someone make a request for a path. They are separate from the path but they are included in the URL.

The format of a query parameter is: name=value. The value can be almost anything, but it is best to avoid “?”. The first parameters is separated from the rest of the URL with “?”, but the second parameter is separated from the first with "ampersand" .

Fragments

A fragments is used to reference a particular part of a page we are looking at and it is separated from the rest of the URL by the “#” sign. They can have other uses as well if they are being used together with JS for example.

When making the request the fragments remain purely in the browser and they are not sent to the server.

NB. We can have both query parameters and fragments, but the fragments must follow the parameter in a URL.

Port
A port is needed in order to establish a connection with a machine. It is part of the URL and it found between the machine name or address and the path. It is separated from the machine name by a column and the default port is 80.
HTTP
HTTP or Hypertext Transfer Protocol is the main protocol of the web and browsers use it in order to connect to the servers.
Browser requests

The request of the URL by a browser begins with a request line, which has three main components: the method, the path and the version. The method is what type of request we are making and the most common ones are GET and POST. The path is what we are actually requesting and the version number used is usually 1.1, but in some cases 1.0 is also used.

For example: GET/ foo HTTP/1.1 – here GET is the method, /foo is the path and 1.1 is the version.

HTTP requests

HTTP requests consist of the request line, host and user-agent, it is followed by headers and all of this information is sent at once when making the request.

One example of a header is the user agent which represents who is making the request and it is one of the most important headers in HTTP requests as it can determine who is trying to hurt a website for example.

HTTP Responses

A response consists of a status line, a status code and the reason phrase, which is the English translation of the status code plus headers.

Example: HTTP /1.1 200 OK - where the version of the status line needs to be the same as that of the request line, 200 is the status code and OK the translation.

Common status codes
  • 200 OK – means that what we were looking for was found
  • 02 FOUND – means that what we are looking for is located somewhere else
  • 404 Not FOUND – what we are looking for was not found
  • 500 Server Error – server broke while we were trying to get what we were looking for

Databases
Database is a program that stores and returns large amount of structured data, but it can also refer to the machine or system of machines running the data.
Why use databases?
Databases are useful for manipulating large chucks of data and answering queries on the data in a short amount of time which is much faster than doing it by hand and without error.
Types of databases
Relational databases or SQL such as MySQL used by Facebook, PostgreSQL used at Reddit, SQLite which cannot be used for working with large amount of data. Other databases include the Google App Engine datastore, Amazon Dynamo, NoSQL.
SQL
SQL or Structured Query Language is not a programming language, but a language used for queries and can be used on a relational database to get data out of it.
Indexes
Indexes increase the speed of databases, but if they are used they make it mandatory to keep them up to date when the database is updated.
ACID
  • Atomicity – all parts of a transaction fail or succeed together
  • Consistency – the database will always be consistent
  • Isolation – no transaction can interfere with another
  • Durability – once a transaction is counted it will not be lost

Overall, ACID is a set of properties that ensure the database is reliable.
